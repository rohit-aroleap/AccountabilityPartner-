## Summary
- Dashboard ideas: **96** across 12 categories
- Prompt engineering ideas: **98** across 14 categories
- Top emergent themes:
  1. **Context richness vs. token bloat** — the prompt builder (`js/prompt.js`, `worker/worker.js:buildCronCheckinPrompt`) passes 14 days of raw data but never summarises trends; smarter context selection would sharpen every AI call.
  2. **Personality system is half-built** — 9 personalities are written and stored, but the Sandbox and Coach button still don't use the per-customer personality; closing this gap is the single biggest quality unlock.
  3. **Operational blind spots** — no alerting on errors, no view of "who hasn't been messaged in N days", no way to A/B test prompt changes; these slow down every iteration cycle.

## Ambitious bets (L effort but transformational)
1. **A/B prompt testing infrastructure** — Champion vs. challenger prompt, random 10% customer allocation, automatic winner selection based on thumbs-up rate. Changes prompt iteration from intuition to evidence. (PE versioning #5)
2. **"Morning run" sweep view** — Sequential workflow that opens each customer needing a check-in, auto-generates, shows for approval, advances to next. Turns a 30-minute multi-tab session into a focused 10-minute workflow. (Multi-customer ops #10)
3. **Full eval suite in Tune AI** — 20 canonical scenarios, automated pass/fail, regression blocking on save. Makes prompt changes safe to ship fast. (Eval methodology #1 + #2)
4. **Per-customer conversation summarisation** — Haiku-powered summary of conversations > 50 messages stored in Firebase. Halves token costs and improves AI recall without hallucination. (Token efficiency #3)
5. **Workout exercise names in prompt context** — Pull the actual exercise names from `exerciseDb` and inject top-3 per recent session. Transforms generic "you trained" into specific "you did Chest Press + Goblet Squat" — the biggest single quality jump for Ferra customers. (Workout knowledge #1)

## Top 10 to ship next
Rank-ordered, mixing both lists, weighing impact / effort / current pain.

1. **Wire per-customer personality to Coach button** (Dashboard AI Quality #1 + PE Language #1) — Personalities are written, stored, but never used by the dashboard-side Coach draft. One afternoon of wiring in `js/prompt.js` + `js/chat.js` delivers the full value of all 9 personalities immediately.

2. **Add Anthropic prompt caching** (PE Token Efficiency #1) — `cache_control: ephemeral` on the system prompt block in `worker.js:callAnthropic`. Estimated 80% reduction in input token cost. Lowest effort, highest ROI of any item on this list.

3. **Inject notes + onboarding answers into prompt context** (PE Context #4 + #9) — Two `lines.push()` additions to `buildFerraPrompt` and `buildCronCheckinPrompt`. Immediately personalises every AI message with data that's already collected but unused.

4. **Opt-out auto-pause** (Dashboard Safety #1) — `detectOptOutKeywords` fires but doesn't set `paused: true`. A missed opt-out that continues sending messages is a trust-destroying (and potentially legal) incident. Fix is one `fbPatch` call.

5. **"No markdown + no filler opener" prompt rules** (PE Output Format #1 + #3) — Two instruction lines added to all three base prompts. Immediately improves message quality and eliminates the most common formatting failures.

6. **Summarise 14-day activity into 3-line trend** (PE Context Engineering #1) — Compute "N sessions last 7 days, streak = X days, last session = Y" and inject before the raw list. Makes the AI's context clearer while reducing tokens.

7. **Sidebar "Sort by needs attention" + Pending draft filter** (Dashboard Multi-customer #1 + #2) — Together these solve the daily workflow problem: knowing who to contact and where drafts are waiting. Two small comparator/filter additions to `customers.js`.

8. **Anti-hallucination examples (ask don't assume)** (PE Anti-hallucination #2) — Add 3 BAD → GOOD worked examples to every base prompt. Pattern-matching against concrete examples reduces hallucination more than abstract rules.

9. **Exercise names in prompt context** (PE Workout Knowledge #1) — Fetch `exerciseDb`, look up exercise names for each day's session, inject top-3 per day. The single largest quality improvement for Ferra customer messages.

10. **Kill switch alert banner + Automation heartbeat** (Dashboard Safety #2 + Automation #10) — A kill switch with no dashboard indicator is a silent failure waiting to happen. The heartbeat tells you the worker is alive. Two small additions, but high operational value.
