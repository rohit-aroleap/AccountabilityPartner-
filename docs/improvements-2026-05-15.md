## Summary
- Dashboard ideas: 88 across 12 categories
- Prompt engineering ideas: 92 across 14 categories
- Top emergent themes:
  1. **Context poverty** — the prompts have rich data structures (onboarding answers, notes, exercise database, monthly summaries, progressive overload counts) that are never injected into AI calls. Closing this gap is the single highest-leverage prompt improvement area.
  2. **Quality feedback loop** — ratings are written to Firebase (`aiRatings`) but never surfaced back to the operator. Building the ratings → prompt improvement loop (ratings tab, bad-draft library, eval suite) is the highest-leverage dashboard category.
  3. **Scale friction** — the architecture handles ~15 customers well but has no tools for operating at 30–100 (no sidebar search, no bulk actions, no unread indicators). Every item in the multi-customer ops and analytics categories addresses this.
